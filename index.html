Style and inspect elements with speed and precision.

DomFlags offers a new way to interact with the DevTools. For the first time ever, you can now create keyboard shortcuts to DOM elements.

DomFlags are like bookmarks for navigating the DOM tree. They help you track important elements so styling can be a breeze. But DomFlags offers even more. The auto-inspect features help you target elemets with pinpoint accuracy so you can reengage and style nodes without losing time.

Every seasoned designer or developer working with the DevTools has struggled with tricky elements that are difficult to inspect, or seem to get lost in the choas of the DOM. I built DomFlags after many days of wishing for a better way to track these precarious elements. Combined with keyboard shortcuts, DomFlags offers a fast and intuitive method to navigate around the DOM.

How to use DomFlags

A DomFlag is created by adding the ‘domflag’ attribute to an element.

<div class="foo"domflag> … </div>

DomFlags can be added to the DOM using the following methods:

  1. Add ‘domflag’ attrbiutes directly in your html documents.
  2. Add ‘domflag’ attributes from the DevTools Elements panel.
  3. Add ‘domflag’ attributes using JavaScript.

The advantage of method #1 is the DomFlags will persist across page reloads. Method #2 is great for a quick one-time use per DevTools session. And method #3 can be especially useful during JavaScript development.

Keyboard shortcuts to DOM elements

Keyboard shortcuts will change the way you use the DevTools. For the first time ever, you can now create keyboard shortcuts to DOM elements. They save you time by giving you a quick way to jump between multiple elements while styling.

  • Option + Shift + D to add/remove a domflag attribute.
  • Option + Shift + [1..3] to inspect a flagged element.
    [4..0] shortcuts can be set in chrome://extensions/configureCommands.

Auto-focusing flagged elements

By default, the DomFlags extension will auto-inspect (focus) the first flagged element in the DOM when:

  1. the DevTools gets opened (cmd + shift + i);
  2. the page is reloaded (with the DevTools open).

These features are useful for targeting an element to ensure the DevTools always reengages that element, regardless of the currently inspected node. They will speed your development by dramatically reducing the time it takes to expand deep nodes, or visually inspect a tricky element.

The auto-inspect features can be toggled from the settings panel.
