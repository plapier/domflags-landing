Style and inspect elements with speed and precision.

DomFlags offers a new way to interact with the DevTools. For the first time ever, you can now create keyboard shortcuts to DOM elements.

DomFlags are like bookmarks for navigating the DOM tree. They help you track important elements so styling can be a breeze. But DomFlags offers even more. The auto-inspect features will help you target elemets with pinpoint accuracy so you can style nodes with speed and precision.

Every seasoned designer or developer working with the DevTools has struggled with tricky elements that are difficult to inspect, or seem to get lost in the choas of the DOM. DomFlags are the solution to my many days of inspecting and styling these precarious DOM elements – especailly DOM elements that are rendered via javascript after the DOM has fully loaded. Users of javascript frameworks like Ember, Backbone, Angular, or any other scripts that swap or load content will find DomFlags incredibly useful.

How to use DomFlags

DomFlags are created by adding the ‘domflag’ attribute to an element.

<div class="foo"domflag> … </div>

DomFlags can be added to the DOM using the following methods:

  1. Add ‘domflag’ attrbiutes directly in your html documents.
  2. Add ‘domflag’ attributes from the DevTools Elements panel.
  3. Add ‘domflag’ attributes using JavaScript.

The advantage of method #1 is the DomFlags will persist across page reloads. Method #2 is great for a quick one-time use per DevTools session. And method #3 can be especially useful during JavaScript development.

Keyboard shortcuts to DOM elements

Keyboard shortcuts will change the way you use the DevTools. For the first time ever, you can now create keyboard shortcuts to DOM elements. You'll save a ton of time quickly jumping between multiple elements during styling.

  • Option + Shift + D to add/remove a domflag attribute.
  • Option + Shift + [1..3] to inspect a flagged element.
    [4..0] shortcuts can be set in chrome://extensions/configureCommands.

Auto-focusing flagged elements

By default, the DomFlags extension will auto-focus (inspect) the first flagged element in the DOM tree when the page is reloaded. This feature is useful for targeting an element and ensuring the DevTools always reengages that element when you reload the page, regardless of the currently inspected node.

DomFlags also gives you the ability to auto-focus a flagged element when the DevTools gets opened (via Cmd + Shift + i). The feature instantly inspects the first flagged element—which can dramatically speed up the time it takes to expand deep nodes, or visually inspect the tricky element.

The auto-focus features can be toggled from the settings panel.
