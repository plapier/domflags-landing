style and inspect elements with speed & precision

DomFlags chrome extension offers a new way to interact with the DevTools. For the first time ever, you can now create keyboard shortcuts to DOM elements.

DomFlags are like bookmarks for navigating the DOM tree. They help you track and navigate to important elements so styling can be a breeze. But DomFlags offers even more. The auto-inspect features help you re-engage elements with pinpoint accuracy to speed your DevTools workflow.

Every seasoned designer or developer working with the DevTools has struggled with tricky elements that are difficult to inspect, or seem to get lost in the chaos of the DOM. And sometimes the DevTools just doesn't do a great job at keeping focus on the elements you're styling. DomFlags was built as a solution to track these important elements. Combined with keyboard shortcuts, DomFlags offers a fast and intuitive method to navigate around the DOM.

How to use DomFlags

A DomFlag is created by adding the ‘domflag’ attribute to an element.

<div class=""domflag> … </div>

DomFlags can be added to the DOM using the following methods:

  1. Add ‘domflag’ attributes directly in your html documents.
  2. Add ‘domflag’ attributes from the DevTools Elements panel.
  3. Add ‘domflag’ attributes using JavaScript.

The advantage of method #1 is the DomFlags will persist across page reloads. Method #2 is great for a quick one-time use per DevTools session. And method #3 can be especially useful during JavaScript development.

Keyboard shortcuts to DOM elements

Keyboard shortcuts will change the way you use the DevTools. For the first time ever, you can now create keyboard shortcuts to DOM elements. They will save you time by giving you quick access to multiple elements during styling.

  • Option + Shift + D to add/remove a domflag attribute.
  • Option + Shift + [1..3] to inspect a flagged element.
    [4..0] shortcuts can be set in chrome://extensions/configureCommands.

Auto-Inspecting flagged elements

By default, the DomFlags extension will auto-inspect (focus) the first flagged element in the DOM when:

  1. the DevTools gets opened (cmd + shift + i);
  2. the page is reloaded (with the DevTools open).

These features are useful for targeting an element to ensure the DevTools always re-engages that element, regardless of the currently inspected node. They will speed your development by dramatically reducing the time it takes to expand deep nodes, or visually inspect a tricky element.

The auto-inspect features can be toggled from the settings panel.
