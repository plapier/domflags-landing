DomFlags help you style and debug elements with speed and precision.

Every seasoned designer or developer working with the DevTools has struggled with tricky elements that are difficult to inspect, or seem to get lost in the choas of the DOM. DomFlags offers a solution. DomFlags are like bookmarks for navigating the DOM tree. They help you track important elements. But DomFlags offers even more. The auto-inspect features can increase your speed and precison while you style or debug flagged elements.

DomFlags are the solution to my many days of inspecting, styling, and debugging precaious DOM elements – especailly DOM elements that are rendered via javascript after the DOM has fully loaded. Users of javascript frameworks like Ember, Backbone, Angular, or any other scripts that swap or load content will find DomFlags incredibly useful.

How to use DomFlags

DomFlags are created by adding the ‘domflag’ attribute to an element.

<div class="foo"domflag> … </div>

DomFlags can be added to the dom using the following methods:

  1. Add ‘domflag’ attrbiutes directly in your html documents.
  2. Add ‘domflag’ attributes from the DevTools Elements panel.

The advantage of method #1 is the DomFlags will persist across page reloads. Method #2 is great for a quick one-time use per DevTools session.

Auto-focusing flagged elements

By default, the DomFlags extension will auto-focus (inspect) the first flagged element in the dom tree when the page is reloaded. This feature is useful for targeting an element and ensuring the DevTools always reengages that element when you reload the page, regardless of the currently inspected node.

Another useful auto-focus feature is for targeting the flagged element when the DevTools is opened (via cmd + shift + i). The feature instantly inspects the flagged element—which can dramatically speed up the time it takes to expand deep nodes, or visually inspect the tricky element.

The auto-focus features can be toggled from the settings panel.

Keep an eye out for more features to be added to the settings panel as development on the extension continues.
